# 手机审批功能使用说明

## 功能概述

手机审批功能为私密备忘录应用添加了完整的审批流程，支持：
- ✅ 创建审批事项
- ✅ 审批通过/拒绝
- ✅ 审批提醒通知
- ✅ 审批统计
- ✅ 审批详情查看

## 快速开始

### 1. 添加审批入口

在应用首页的快捷入口卡片中，已经添加了"审批中心"入口，显示待审批数量徽章。

### 2. 创建审批

**操作路径：**
- 首页 → 点击"审批中心"卡片
- 点击右下角"新建审批"按钮
- 填写审批标题、审批人、备注信息
- 点击"创建"按钮

**审批信息：**
- **审批标题**：必填，简短描述审批事项
- **审批人**：必填，审批人的姓名
- **备注**：选填，详细的审批说明

### 3. 审批流程

**待审批事项显示：**
- 在审批中心首页，默认显示所有待审批事项
- 支持筛选：全部/待审批/已通过/已拒绝
- 每条审批显示申请人、创建时间、备注等信息

**审批操作：**
- 点击"通过"按钮 → 弹出确认对话框 → 确认通过
- 点击"拒绝"按钮 → 弹出确认对话框 → 确认拒绝
- 点击"删除"按钮 → 弹出确认对话框 → 确认删除
- 点击"详情"图标 → 查看完整审批信息

### 4. 审批通知

**通知类型：**
1. **新审批通知**：创建新审批时，立即弹出通知
2. **待审批提醒**：应用启动时，检查是否有待审批事项，显示通知和SnackBar
3. **审批结果通知**：审批通过/拒绝后，发送相应通知

**通知设置：**
- 需要在应用首次启动时授权通知权限
- 应用内会自动请求通知权限
- 通知权限可在系统设置中管理

### 5. 审批统计

**统计信息：**
- 待审批数量：显示在首页审批中心卡片徽章
- 已通过数量：可在审批中心筛选查看
- 已拒绝数量：可在审批中心筛选查看

## 技术实现

### 核心组件

1. **ApprovalItem** (数据模型)
   - 审批ID、标题、申请人、审批人
   - 审批状态（待审批/已通过/已拒绝）
   - 创建时间、审批时间、备注

2. **ApprovalBloc** (状态管理)
   - 使用Flutter Bloc模式
   - 管理审批数据的增删改查
   - 处理审批状态变更

3. **ApprovalCenterScreen** (UI界面)
   - 审批列表展示
   - 审批通过/拒绝操作
   - 筛选功能
   - 详情弹窗

4. **ApprovalNotificationService** (通知服务)
   - 本地通知推送
   - 通知权限管理
   - 定时通知支持

5. **ApprovalNotificationHelper** (通知助手)
   - 自动检查待审批事项
   - 创建审批时发送通知
   - 审批结果通知

### 数据库

**approvals 表结构：**
```sql
CREATE TABLE approvals (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  applicant TEXT NOT NULL,
  approver TEXT NOT NULL,
  status TEXT NOT NULL,
  createdAt TEXT NOT NULL,
  notes TEXT,
  approvedAt TEXT,
  approvedBy TEXT
)
```

**字段说明：**
- `id`：审批唯一标识
- `title`：审批标题
- `applicant`：申请人
- `approver`：审批人
- `status`：审批状态（pending/approved/rejected）
- `createdAt`：创建时间（ISO 8601格式）
- `notes`：备注信息
- `approvedAt`：审批时间
- `approvedBy`：审批人

## 使用场景

### 场景1：请假审批
1. 申请人创建审批："2026年清明节请假审批"
2. 审批人：张经理
3. 审批人收到通知 → 查看详情 → 通过审批

### 场景2：费用报销
1. 申请人创建审批："3月份差旅费用报销"
2. 审批人：李财务
3. 审批人查看详情 → 核对票据 → 拒绝审批
4. 申请人收到拒绝通知 → 修改后重新提交

### 场景3：物品借用
1. 申请人创建审批："借用公司笔记本电脑"
2. 审批人：王主任
3. 审批人收到通知 → 查看借用时间 → 通过审批

## 注意事项

1. **权限要求**
   - 应用需要通知权限才能发送提醒
   - 首次使用时会自动请求权限
   - 可在系统设置中管理权限

2. **数据安全**
   - 审批数据存储在加密数据库中
   - 所有审批记录本地存储，不上传云端
   - 支持生物识别认证保护

3. **通知设置**
   - 需要在应用内授权通知权限
   - 通知权限被拒绝后，功能受限
   - 可在应用设置中重新授权

4. **性能优化**
   - 审批列表支持分页加载
   - 使用 IndexedStack 优化导航性能
   - 定期清理过期审批记录

## 常见问题

### Q1: 为什么收不到审批通知？
**A:** 请检查以下几点：
1. 是否授权了通知权限
2. 系统通知设置是否开启
3. 应用是否在后台运行
4. 检查设备是否允许应用发送通知

### Q2: 如何批量审批？
**A:** 当前版本不支持批量审批，需要逐条查看并操作。后续版本会添加批量操作功能。

### Q3: 审批记录会同步到云端吗？
**A:** 不会。所有审批记录都存储在本地加密数据库中，确保数据安全。

### Q4: 如何删除审批记录？
**A:**
1. 进入审批中心
2. 找到要删除的审批
3. 点击删除图标
4. 确认删除

### Q5: 审批统计在哪里查看？
**A:**
- 待审批数量：显示在首页审批中心卡片徽章
- 已通过/已拒绝：可在审批中心筛选查看
